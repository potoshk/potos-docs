# Contributor Guide

Welcome to the POTOS contributor documentation. This section contains technical guides for developers who want to contribute to the POTOS node.

## About This Section

This section is for developers who want to dive into POTOS node implementation, participate in node development, or contribute to the underlying technology. The guides cover the full workflow from building from source to running a local chain, so you can set up your development environment and start contributing.

## Documentation

### [Build from Source](./compile_from_source.md)

Learn how to compile FISCO BCOS (the framework POTOS is built on) from source. The guide includes dependency installation and build steps for major operating systems.

### [Build a Local Chain](./build_chain.md)

Use the build script to create a local FISCO BCOS blockchain network for development and testing.

## How to Contribute

To contribute to POTOS, we recommend:

1. Read the [Community Contribution Guide](../community/contribute.md) to understand the contribution process.
2. Explore the [GitHub repository](https://github.com/FISCO-BCOS/FISCO-BCOS) to learn the project structure.
3. Use the technical docs in this section to set up your development environment.
4. Join the [community](../community/index.md) to connect with other contributors.
